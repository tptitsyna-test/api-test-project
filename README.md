# API-тесты DummyJSON

Набор API-проверок для сценария авторизации и работы с корзинами https://dummyjson.com
Проект выполнен на Playwright + TypeScript

Тестовые учётные данные загружаются из локального `.env` - данные взяты из публичной документации DummyJSON
Файл `.env` добавляется в Git, но в реальных проектах был бы добавлен в .gitignore

ссылка на Git-репозиторий https://github.com/tptitsyna-test/api-test-project

## Мой опыт

- Ручное тестирование: опыт функционального, регрессионного и исследовательского тестирования
- Нагрузочное тестирование: опыт подготовки и запуска нагрузочных сценариев с использованием JMeter, Postman
- Тестирование безопасности: опыт проверок авторизации, аутентификации, валидации входных данных и доступа к ресурсам

## Инструменты

- Node.js
- TypeScript
- Playwright Test
- npm

## Запуск

Требуется Node.js 20 или новее

```bash
npm ci
cp .env.example .env
npm test
```

## Реализованные проверки

### Авторизация

1. `POST /auth/login` с корректными данными: статус, данные пользователя и наличие токенов
2. `POST /auth/login` с неверным паролем: статус 400 и сообщение об ошибке
3. `POST /auth/login` без username: статус 400 и сообщение об обязательных полях
4. `POST /auth/login` без password: статус 400 и сообщение об обязательных полях
5. `GET /auth/me` с Bearer-токеном: статус и данные текущего пользователя
6. `GET /auth/me` без токена: статус 401
7. `GET /auth/me` с некорректным токеном: статус 401

### Корзина

8. `GET /carts/user/{userId}`: получение корзин и проверка их принадлежности пользователю
9. `GET /carts/user/{userId}`: пустой список для пользователя без корзин - skipped потому что в текущих данных DummyJSON у каждого пользователя есть корзина
10. `GET /carts/{cartId}`: получение существующей корзины и проверка расчётов
11. `GET /carts/{cartId}` с отрицательным ID: статус 404
12. `POST /carts/add`: создание корзины с одним товаром
13. `POST /carts/add`: создание корзины с несколькими единицами одного товара
14. `POST /carts/add`: создание корзины с несколькими разными товарами
15. `POST /carts/add` без userId: статус 400
16. `POST /carts/add` с пустым products: статус 400
17. `POST /carts/add` с неизвестным productId: товар не добавляется
18. `PATCH /carts/{cartId}`: замена состава корзины
19. `PATCH /carts/{cartId}` с 'merge: true': добавление товара без удаления исходных позиций
20. `PATCH /carts/{cartId}`: удаление товара из корзины (установка количества товара в 0)
21. `DELETE /carts/{cartId}`: удаление существующей корзины
22. `DELETE /carts/{cartId}` для несуществующей корзины: статус 404
23. `GET /carts/{cartId}` для несуществующей корзины: статус 404

Для товаров проверяются расчёты total, discountedPrice или discountedTotal, а также итоговые значения корзины

## Список дефектов/вопросов

1. `GET /auth/me` возвращает поле `password` со значением `emilyspass`
Для реального сервиса это создавало бы риск раскрытия учётных данных
2. На запрос `POST /carts/add` с `quantity: 0` сервис ответил `201` и вернул `quantity: 1` вместо ошибки
Документация не задаёт допустимый диапазон и ожидаемый статус ошибки
3. Неизвестный `productId` не отклоняется: API отвечает `201` и создаёт пустую корзину
4. CRUD-операции являются симуляцией и не позволяют проверить фактическое сохранение данных
